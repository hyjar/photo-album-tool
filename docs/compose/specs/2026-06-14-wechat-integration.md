# 微信接入设计规范

## 概述

将MiMoCode摄影集排版工具接入微信个人号，用户可以通过微信好友对话与MiMoCode交互，使用全部功能（代码问答、项目管理、代码审查等）。

## 架构设计

### 技术栈
- **前端**：现有Vite项目（保持不变）
- **后端**：Node.js + Express + Wechaty
- **MiMoCode调用**：通过子进程调用mi命令行工具

### 系统架构
```
用户微信 → 微信服务器 → Wechaty服务 → MiMoCode → 返回结果 → 微信
```

### 模块划分
1. **微信机器人模块** (`server/wechat-bot.js`)
   - 处理微信消息接收和发送
   - 管理用户会话状态

2. **MiMoCode调用模块** (`server/mimo-runner.js`)
   - 封装mi命令行调用
   - 处理命令执行和结果解析

3. **会话管理模块** (`server/session-manager.js`)
   - 管理用户对话上下文
   - 维护对话历史

4. **主服务器** (`server/index.js`)
   - Express服务器
   - 路由配置
   - WebSocket支持（可选）

## 功能范围

### 支持的命令
- `/help` - 显示帮助信息
- `/code <code>` - 审查代码
- `/ask <question>` - 问答
- `/task [list|create|update|delete]` - 任务管理
- `/project [list|create|switch]` - 项目管理
- `/export` - 导出摄影集
- `/status` - 查看系统状态

### 消息类型支持
- 文本消息
- 图片消息（用于代码审查）
- 文件消息（用于项目文件）

## 安全考虑

1. **用户认证**
   - 白名单机制：只有授权用户可以使用
   - 可选：微信OAuth认证

2. **命令过滤**
   - 禁止危险操作（rm -rf, 格式化等）
   - 限制命令执行时间

3. **数据安全**
   - 不存储敏感信息
   - 会话历史定期清理

## 部署要求

1. **服务器要求**
   - Node.js 18+
   - 稳定的网络连接
   - 微信账号（用于登录）

2. **依赖服务**
   - 微信账号（需要手动扫码登录）
   - MiMoCode命令行工具（已安装）
